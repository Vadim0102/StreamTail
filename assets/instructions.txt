🔑 ОБЩИЙ REDIRECT URI для всех платформ: http://localhost:19234/callback

🎮 1. TWITCH (Регистрация Developer-приложения):
   • Перейдите на консоль разработчика Twitch: https://dev.twitch.tv/console
   • Войдите в аккаунт и нажмите кнопку "Register Your Application" (Зарегистрировать приложение).
   • Заполните форму:
     - Name: Введите уникальное имя приложения (например, "StreamTail - MyChannel").
     - OAuth Redirect URLs: Введите строго http://localhost:19234/callback и нажмите кнопку "Add".
     - Category: Выберите "Application Integration" или "Other".
   • Нажмите "Create" (Создать).
   • В списке зарегистрированных приложений нажмите "Manage" (Управлять) напротив вашего приложения:
     - Скопируйте строку "Client ID".
     - Нажмите кнопку "New Secret" (Создать новый секрет) и обязательно скопируйте сгенерированный "Client Secret" (он показывается только один раз).
   • Перенесите полученные Client ID и Client Secret в поля настроек Twitch в StreamTail, после чего выполните авторизацию во вкладке "Авторизация".

🎥 2. YOUTUBE / GOOGLE CLOUD (Регистрация OAuth-клиента):
   • Перейдите в Google Cloud Console: https://console.cloud.google.com/
   • Создайте новый проект (выпадающий список в верхнем левом углу -> "New Project").
   • В строке поиска сверху введите "YouTube Data API v3", откройте его страницу и нажмите "Enable" (Включить).
   • Настройте экран согласия (OAuth consent screen) на боковой панели:
     - Выберите тип "External" (Внешний) и нажмите "Create".
     - Заполните обязательные поля: название приложения, email поддержки и контактную информацию разработчика. Нажмите "Save and Continue".
     - В разделе "Test users" (Тестовые пользователи) обязательно добавьте ваш личный Google-аккаунт, на котором вы ведете стримы. Без этого Google заблокирует авторизацию, так как приложение находится в статусе тестирования. Нажмите "Save and Continue".
   • Перейдите в раздел "Credentials" (Учетные данные) на боковой панели:
     - Нажмите "Create Credentials" -> "OAuth client ID".
     - Application type: Выберите "Web application" (Веб-приложение).
     - Name: Укажите имя (например, "StreamTail YouTube Client").
     - Authorized redirect URIs: В разделе "Разрешенные URI перенаправления" нажмите "Add URI" и вставьте:
       http://localhost:19234/callback
     - Нажмите "Create".
   • Скопируйте выданные "Client ID" и "Client Secret" и вставьте их в настройки YouTube в StreamTail.

🌐 3. VK LIVE (Создание приложения или импорт сессии):
   • Owner ID: Введите имя вашего канала в VK Видео (последняя часть ссылки на ваш канал, например, "vadimzaa" или цифровой ID блога).
   • Способ А (РЕКОМЕНДУЕМЫЙ - Быстрый импорт токена без создания приложений):
     - Откройте сайт https://live.vkvideo.ru в вашем обычном браузере, где выполнен вход под вашим стримерским аккаунтом.
     - Нажмите F12 (или Ctrl+Shift+I) и перейдите во вкладку "Application" (Приложение / Хранилище).
     - Слева раскройте ветку "Local Storage" (Локальное хранилище) и выберите адрес "https://live.vkvideo.ru".
     - Справа найдите ключ с именем "auth" и скопируйте всю его длинную JSON-строку (она начинается с {"accessToken": ...}).
     - Вставьте скопированный JSON целиком в поле "Токен" настроек VK LIVE. Данные Client ID / Client Secret в этом случае заполнять не требуется!
   • Способ Б (Создание своего приложения VK ID):
     - Перейдите на портал разработчиков VK: https://dev.live.vkvideo.ru/apps
     - Создайте новое приложение и пропишите Redirect URI: http://localhost:19234/callback.
     - Перенесите выданные ID приложения (Client ID) и Защищенный ключ (Client Secret) в настройки StreamTail для прохождения классического OAuth.

🏆 4. GOODGAME (Регистрация приложения разработчика):
   • Перейдите на сайт GoodGame.ru, откройте настройки своего профиля и выберите вкладку "OAuth" / "Разработчикам".
   • Нажмите кнопку "Добавить новое приложение".
   • Укажите параметры:
     - Название приложения: StreamTail Client
     - Redirect URI: http://localhost:19234/callback
   • Сохраните настройки и скопируйте предоставленные Client ID и Client Secret в настройки GoodGame в StreamTail.

🇷🇺 5. RUTUBE (Куки Студии вещания):
   • ID Канала: Укажите имя вашего публичного канала (для беспрепятственного чтения статуса и онлайна без авторизации).
   • ID Стрима: Скопируйте уникальный хэш трансляции из адресной строки вашей Студии вещания (например, "f290551824869de96ec29760e731385d" из URL https://studio.rutube.ru/stream/f290551824869de96ec29760e731385d).
   • Токен: Экспортируйте ваши куки с сайта studio.rutube.ru (через расширение EditThisCookie или Cookie Quick Manager в формате JSON или Netscape) и вставьте текст полностью в поле "Токен". Плагин автоматически выделит сессию и CSRF-токены защиты.

💚 6. KICK (Регистрация приложения в кабинете разработчика):
   • Имя канала: Введите slug вашего канала.
   • Способ А (ОФИЦИАЛЬНЫЙ API):
     - Зарегистрируйте приложение в личном кабинете разработчика Kick: https://kick.com/settings/developer
     - Укажите Redirect URI: http://localhost:19234/callback
     - Скопируйте полученный Bearer Token, Client ID и Client Secret. Вставьте их в поля настроек Kick в StreamTail.
   • Способ Б (НЕОФИЦИАЛЬНЫЙ API):
     - Откройте Kick.com, войдите в аккаунт, откройте консоль разработчика браузера (F12) -> вкладка Network (Сеть).
     - Найдите любой исходящий запрос к сайту, скопируйте значение заголовка "Cookie" и вставьте его в поле "Токен" в StreamTail.
